# MolTrace — Backend-surfacing audit (Phase 2)

_Inventory of UI that surfaces the backend (raw IDs/JSON, internal enums, dev-JSON panels, raw status). Generated by an 8-cluster fan-out audit; drives Phases 3–7 of `app_simplification_plan.md`. FE-only; no contract changes._

**96 findings** — 36 high · 46 medium · 14 low.

## Rollup by phase

| Phase | Findings | What it fixes |
|---|---|---|
| Phase 3 — entity pickers | 37 | raw integer-ID / FK inputs + JSON textareas → searchable pickers / structured editors |
| Phase 4 — read-only server values | 6 | content-hash / server-owned fields → display, not input |
| Phase 5 — vocabulary & status | 31 | snake_case labels + raw status strings → human labels + colored badges |
| Phase 6 — developer-mode gate | 18 | "Developer JSON" tabs / raw-payload dumps → behind an opt-in toggle |
| Phase 7 — de-form | 4 | endpoint-shaped forms → fold into workflow or move to API |
