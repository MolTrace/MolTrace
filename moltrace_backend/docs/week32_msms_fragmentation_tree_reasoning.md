# Week 32: Processed MS/MS Fragmentation-Tree + Diagnostic Neutral-Loss Reasoning

## Purpose

Week 32 upgrades the processed MS/MS layer from peak annotation into interpretable fragmentation reasoning.

The goal is to connect:

```text
precursor ion -> fragment ion -> subfragment ion
```

using:

- observed processed MS/MS peak tables;
- precursor m/z;
- adduct;
- exact precursor mass consistency;
- common neutral losses;
- candidate-specific chemical plausibility checks;
- simple single-bond fragment hypotheses;
- contradiction flags requiring human review.

## Supported Evidence

The module accepts processed centroid MS/MS peak tables:

```text
m/z,intensity
47.04914,10
29.03858,100
```

It intentionally works from processed peak-list views. Use the Week 35 LC-MS/MS import bridge to extract those views from mzML/mzXML or source files.

## Diagnostic Losses

The current transparent rule set checks:

```text
H2O
NH3
CO
C2H4
CH3OH
H2S
HCl
C2H2O
CO2
CH3COOH
SO2
HBr
```

## Candidate-Specific Contradictions

Examples:

- HCl loss in a candidate without chlorine.
- HBr loss in a candidate without bromine.
- NH3 loss in a candidate without nitrogen/amine/amide support.
- CO2 loss in a candidate without carboxyl/ester/carbonate support.
- SO2 loss in a candidate without sulfur/sulfone/sulfonyl support.
- Precursor m/z inconsistent with candidate formula and adduct.

## Scientific Boundary

MS/MS fragmentation-tree evidence supports or weakens a candidate, but it does not prove final connectivity or stereochemistry. Final interpretation should combine HRMS, adduct/isotope evidence, 1H/13C/2D NMR, predicted NMR matching, and human review.

## Known Limitations

This is not a complete fragmentation simulator. It consumes processed peak-list
views, including those generated by the Week 35 LC-MS/MS import bridge, but it
does not yet:

- directly interpret proprietary vendor raw data;
- perform full mzML transformation or deconvolution inside the tree engine;
- use collision-energy series;
- perform chromatographic deconvolution;
- build exact mechanistic fragmentation trees;
- run database search;
- generate unknown molecular structures.
