# Raw FID Prompt 1/2 Manual Promotion Design

This document defines the only acceptable path for moving the Prompt 1/2
Bruker/Varian FID reader, phase correction, and baseline correction from
metadata-only diagnostics toward visible SpectraCheck raw-FID behavior.

Current policy: Prompt 1/2 is not active for plotted spectra, peak markers,
phase correction, baseline correction, legends, integration, qNMR, or
processed-spectrum behavior. The existing SpectraCheck raw-FID and
processed-spectrum pipelines remain authoritative.

## Non-negotiable Invariants

- Processed 1H and 13C spectra must remain untouched by raw-FID promotion work.
- Raw-FID upload vault records must remain immutable. Promotion may create
  derived metadata, but it must never rewrite uploaded archives.
- Regentry and Repho behavior must not change as a side effect of
  SpectraCheck raw-FID work.
- The existing industry-standard visible raw-FID spectrum remains the fallback at every
  stage.
- Unknown or misspelled runtime modes must resolve to `legacy`.
- Prompt 1/2 sidecar output may not drive visible `x/y`, peak markers, legends,
  phase, baseline, integration, or analysis tables until a separate promotion
  implementation is reviewed and tested.
- A failed Prompt 1/2 diagnostic must degrade to metadata warnings, not failed
  user uploads, unless the user explicitly requested a diagnostic report.

## Existing Gates

The current code already exposes a reporting-only readiness object:
`raw_fid_prompt_activation_readiness_v1`.

Before any promotion branch is opened, the CI artifact generated by:

```bash
PYTHONPATH=src uv run moltrace-raw-fid-sidecar-report \
  --limit 20 \
  --include-varian \
  --quiet \
  --promotion-gate \
  --output-dir reports/raw_fid_prompt_manual_promotion_gate
```

must report no failed gates and no review-required gates for representative
Bruker and Varian/Agilent fixtures. The required gates are:

- Prompt sidecar available for every fixture.
- Prompt peak count within fixture reference tolerance.
- ppm-axis endpoint agreement within 0.01 ppm.
- processed Bruker sidecar peak ppm references matched within tolerance when
  available.
- phase-angle delta within 5 degrees.
- baseline method matches nucleus defaults: 1H uses Bernstein, 13C uses
  Whittaker.
- deterministic 64-character fingerprint hash present.
- Prompt sidecar generation completes within 3000 ms per fixture.
- fixture diagnostics do not mark any row safe for runtime activation.

Passing those gates does not activate Prompt 1/2. It only permits a manual
promotion design review.

## Required Fixture Coverage

Promotion review must include:

- At least 20 Bruker NMRShiftDB2 FIDs with processed sidecars when available.
- Varian/Agilent fixture coverage from the nmrglue example set.
- A 100-spectrum externally-processed reference set for phase and baseline
  comparison before any visible phase/baseline change is considered.
- Representative in-house carbohydrate and aminoglycoside spectra, including
  tobramycin derivative examples, so the peak-picking and proton/carbon
  inventory logic is not tuned only for generic organic spectra.
- Negative-control archives with missing or malformed vendor files.

## Promotion Stages

### Stage 0: Current State

Prompt 1/2 runs only as hidden or admin-visible metadata. It may be displayed as
QA context, but the visible raw-FID spectrum, peak markers, and analysis remain
legacy.

Required checks:

```bash
./scripts/run_prompt_sidecar_guardrails.sh
PYTHONPATH=src uv run pytest -q tests/test_fid.py tests/test_nmr_frontend_upload_api.py
pnpm --dir moltrace_frontend test:spectracheck-sidecar
```

### Stage 1: Shadow Comparison

Add a separate admin-only comparison artifact that renders Prompt 1/2 output
beside the legacy result. Do not replace the user-visible trace. Do not feed
Prompt 1/2 peaks into evidence tables.

Required extra checks:

- legacy and Prompt 1/2 traces are both available in diagnostics.
- production user routes still render only the legacy trace.
- no processed-spectrum endpoint includes Prompt 1/2 raw-FID fields.

### Stage 2: Explicit Per-Session Preview Candidate

Allow an authenticated test user to opt into a candidate Prompt 1/2 raw-FID
preview for one upload/session. This must use a new explicit mode name; the
existing `hybrid_metadata` mode must remain metadata-only.

Required extra checks:

- visible output includes a clear candidate label.
- leaving the session or disabling the toggle returns to legacy output.
- preview, process, evidence tables, and downloads identify which pipeline
  produced each derived value.
- upload/session state is not cleared by navigation.

### Stage 3: Limited Canary

Enable Prompt 1/2 candidate rendering only for selected internal workspaces.
Canary results must be compared against legacy output and recorded as analytics
events without changing default behavior for other users.

Required extra checks:

- rollback to `legacy` can be done by configuration only.
- analytics distinguish legacy, sidecar, candidate, and fallback paths.
- dashboard, Regentry, and Repho regressions remain green.

### Stage 4: Default Promotion

Default promotion is allowed only after Stage 3 has shown stable scientific and
UI behavior across fixture and internal testing. The promotion must be a
dedicated pull request with release notes, screenshots, fixture reports, and a
rollback plan.

## Pull Request Checklist

Every promotion pull request must show:

- `./scripts/run_prompt_sidecar_guardrails.sh` passes.
- full backend raw-FID tests pass.
- frontend SpectraCheck viewer/session tests pass.
- fixture report JSON/CSV artifacts are attached.
- before/after screenshots for raw 1H and 13C FID preview/process are attached.
- processed 1H and 13C screenshots are unchanged.
- peak count, ppm alignment, phase, baseline, and runtime gates are summarized.
- failure fallback to legacy is tested.
- Regentry and Repho smoke paths are unaffected.

## Rollback Plan

The fallback path must remain deployable at all times:

```text
MOLTRACE_RAW_FID_PIPELINE=legacy
```

If any promoted stage causes distorted spectra, missing uploads, incorrect peak
picking, slow analysis, or dashboard/session persistence regressions, revert the
runtime mode to `legacy` first, then debug using the sidecar/provenance
artifacts. Do not patch by weakening the spectrum constraints.
